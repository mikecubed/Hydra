# Implementation Plan: Agent Plugin Interface

**Date**: 2026-03-09 | **Spec**: [spec.md](./spec.md) | **Research**: [research.md](./research.md)

## Summary

Hydra hard-codes per-agent behavior across 9 files using agent-name string checks. This plan moves all per-agent behavior (output parsing, error patterns, economy model, model ownership, quota verification, task rules, read instructions) into each agent's definition object in `hydra-agents.mjs`. The executor and all call-sites become data-driven. No new files, no external plugin loading, no breaking API changes.

## Research Findings

See [research.md](./research.md) for full detail. Key decisions:

### Decision 1: Gemini routing — patch `invoke.headless` at executor module scope

**Problem:** `hydra-agents.mjs` cannot import `executeGeminiDirect` from `agent-executor.mjs` (circular import). Gemini's `invoke.headless` is defined in `hydra-agents.mjs` but needs to call a function that only exists in `agent-executor.mjs`.

**Decision:** At `agent-executor.mjs` module scope (synchronously, after imports, before `executeAgent` is defined), overwrite `gemini.invoke.headless` to be a function that calls `executeGeminiDirect` and returns its result wrapped in a well-known sentinel that the executor detects. This is a one-time synchronous patch — no circular import, no lazy loading.

```javascript
// agent-executor.mjs — module scope, after imports
{
  const _g = getAgent('gemini');
  if (_g?.invoke) {
    _g.invoke.headless = async (prompt, opts) => {
      // Sentinel: executor checks for this and calls executeGeminiDirect directly.
      // This keeps the workaround encapsulated here; the public plugin interface stays clean.
      return ['__gemini_direct__', { prompt, opts }];
    };
  }
}
```

In `executeAgent()`, after `agentDef.invoke.headless(...)` is called:

```javascript
const [cmd, invokeArgs] = await agentDef.invoke.headless(prompt, headlessOpts);
if (cmd === '__gemini_direct__') {
  try {
    return await executeGeminiDirect(invokeArgs.prompt, invokeArgs.opts);
  } finally {
    _hubCleanup();
  }
}
```

> The sentinel string `'__gemini_direct__'` is **private** to `agent-executor.mjs`. It is not part of the public plugin interface and never appears in `hydra-agents.mjs`. When the Gemini CLI bug is fixed, delete the module-scope patch and the sentinel check — only those two lines change. This is the "transparent delegation" described in the original plan.

### Decision 2: `parseOutput` replaces both executor parsing and metrics parsing

`hydra-metrics.mjs` currently parses Claude's JSON stdout directly (primary token extractor). `agent-executor.mjs` only parses Codex output today. After Phase 2, the executor calls `agentDef.parseOutput(rawOutput)` for all agents and includes `tokenUsage` in the result. `hydra-metrics.mjs` per-agent blocks become unreachable and are deleted in Phase 3. The `result.tokenUsage` path in `recordCallComplete()` (already present) is sufficient.

### Decision 3: Economy model in `hydra-actualize.mjs`

The 3-way ternary (lines 384–392) maps directly to `agentDef.economyModel(budgetCfg)`. Codex preserves `budgetCfg.handoffModel || 'o4-mini'`; Claude uses `'claude-sonnet-4-5-20250929'`; Gemini uses `'gemini-3-flash-preview'`; local returns `null`.

### Decision 4: `hydra-evolve.mjs` special case

The original plan table says _"Replace Codex error handling special case with `agentDef.errorPatterns` check."_ After inspecting the actual code (lines 469–494), the guard wraps `detectCodexError()` which parses JSONL-specific error events — it is fundamentally a JSONL output concern, not an error-pattern concern.

**Decision:** Use `agentDef?.features?.jsonOutput` as the guard, not an `errorPatterns` emptiness check. Rationale: `detectCodexError()` parses structured JSONL; it should run for any agent that emits JSONL output, not just those with non-empty `errorPatterns` (since all agents now have `errorPatterns`, even if empty `{}`). `detectCodexError()` itself is preserved unchanged.

This is a deliberate, documented deviation from the original plan's table entry. The semantic intent is the same (generalize away from the name `'codex'`); the mechanism is more accurate.

### Decision 5: `hydra-audit.mjs`

`getAgentCommand()` switch is replaced by `getAgent(agent)?.invoke.nonInteractive(prompt)` for the base command, and `getAgent(agent)?.economyModel()` for economy model injection.

**Known residual:** The economy model arg format differs per CLI (`--model` for Claude, `-m` for Codex, none for Gemini). This is a CLI flag formatting detail — it belongs in the `invoke` layer, not in agent behavior. Two options:

- **(a) Accepted residual (chosen for this refactor):** Keep a small arg-format conditional in `getAgentCommand()`, documented as CLI formatting only, not agent logic. This keeps the refactor surgical.
- **(b) Future improvement:** Add `invoke.withEconomyModel(baseArgs, modelId)` to the invoke interface to make `getAgentCommand()` fully data-driven.

Option (a) is chosen for this refactor. Hardcoded model IDs (`claude-haiku-4-5-20251001`, `o4-mini`) are still removed; only the flag format (`-m` vs `--model`) remains as a small conditional.

## Interface Contracts

See [`contracts/plugin-interface.md`](./contracts/plugin-interface.md) for the full typed interface.

### Summary

Every registered agent (after `registerAgent()`) is guaranteed to have:

```javascript
{
  // Feature flags
  features: {
    executeMode: 'spawn' | 'api',   // routing in executor
    jsonOutput: boolean,             // stdout is JSON/JSONL
    stdinPrompt: boolean,            // prompt via stdin vs -p flag
    reasoningEffort: boolean,        // supports --reasoning-effort
  },

  // Methods (all have defaults applied by registerAgent())
  parseOutput(stdout, opts?)       → { output, tokenUsage, costUsd }
  errorPatterns                    → { authRequired?, rateLimited?, quotaExhausted?, networkError? }
  modelBelongsTo(modelId)          → boolean
  quotaVerify(apiKey)              → Promise<{ verified, status?, reason? } | null>
  economyModel(budgetCfg?)         → string | null
  readInstructions(instructionFile) → string
  taskRules                        → string[]
}
```

**Defaults applied by `registerAgent()` when fields are absent:**

| Field                      | Default                                                       |
| -------------------------- | ------------------------------------------------------------- |
| `features.executeMode`     | `'api'` if `customType === 'api'`, else `'spawn'`             |
| `features.jsonOutput`      | `false`                                                       |
| `features.stdinPrompt`     | `false`                                                       |
| `features.reasoningEffort` | `false`                                                       |
| `parseOutput(stdout)`      | `() => ({ output: stdout, tokenUsage: null, costUsd: null })` |
| `errorPatterns`            | `{}`                                                          |
| `modelBelongsTo()`         | `() => false`                                                 |
| `quotaVerify()`            | `async () => null`                                            |
| `economyModel()`           | `() => null`                                                  |
| `readInstructions(f)`      | `` (f) => `Read ${f} first.` ``                               |
| `taskRules`                | `[]`                                                          |

After all imports, before `executeAgent` is defined:

```javascript
// Patch gemini's headless invoke to transparently delegate to executeGeminiDirect.
// This is a module-scope one-time patch that avoids a circular import between
// hydra-agents.mjs and agent-executor.mjs. Gemini's headless definition in
// hydra-agents.mjs is intentionally left as a stub; this overwrites it.
{

    _g.invoke.headless = async (prompt, opts) => {
```

#### 3h. `hydra-audit.mjs` — replace `getAgentCommand()` hardcodes

Economy model IDs are removed from the switch. The CLI flag format (`--model` vs `-m`) differs per agent but is a formatting detail, not agent behavior — it stays as a small documented conditional (see Decision 5).

```javascript
function getAgentCommand(agentName, prompt, economy) {
  const agentDef = getAgent(agentName);
  if (!agentDef) throw new Error(`Unknown agent: ${agentName}`);
  const [cmd, baseArgs] = agentDef.invoke.nonInteractive(prompt, {});
  if (economy) {
    const econModel = agentDef.economyModel();
    if (econModel) {
      // CLI flag format is a formatting detail per agent CLI, not agent behavior:
      // Claude uses --model; Codex uses -m; Gemini has no API-key economy mode.
      const modelArgs =
        agentName === 'claude'
          ? ['--model', econModel]
          : agentName === 'codex'
            ? ['-m', econModel]
            : [];
      return { cmd, args: [...baseArgs, ...modelArgs] };
    }
  }
  return { cmd, args: baseArgs };
}
```
